# Serkonia

## Architecture notes

См. файл `ARCHITECTURE_REVIEW.md` с аудитом текущей структуры и пошаговым планом перехода к сервисному подходу.

## Единый файл изменений (Overlay-подход)

Проект хранит базовые данные в модульных JSON:
- `data/markers.json`
- `data/timeline.json`
- `data/archive.json`

Для редактора добавлен единый файл изменений `data/changes.json`:
- содержит только дельту (изменения), а не полный снимок мира;
- применяется поверх базовых данных при загрузке;
- может быть экспортирован одной кнопкой как `world-changes.json`.

Это позволяет:
1. сохранять модульную структуру контента;
2. скачивать только один файл после сессии редактирования;
3. подготовить инфраструктуру к будущему редактору карты/архива/таймлайна.

## Быстрое включение режима редактора

Горячая клавиша редактора доступна как скрытая функция без дополнительной авторизации:
- `Ctrl + Shift + \`` переключает edit-mode в любой момент.
